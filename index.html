Het verhaal

Een rivier die Brussel
vergat maar niet vergat

"Ik gaf de stad haar naam, haar water, haar bier, haar leven — en zij begroef mij."

De Zenne was de levensader van Brussel. "Bruocsella" — de naam van de stad — betekent nederzetting in het broekland. Vissers werkten haar oevers. Brouwers putten haar water voor het unieke Lambic-bier. De Zenne was Brussel.

Maar de industrialisatie veranderde alles. Fabrieksafval vergiftigde haar waters. Cholera-epidemieën eisten duizenden levens. In 1867 nam burgemeester Anspach een radicale beslissing: de Zenne verdwijnt. Overwelfd. Begraven. Vergeten.

Vandaag stroomt ze nog steeds — onzichtbaar, stil, maar levend — 11 kilometer door de onderbuik van de stad. Ze wacht. En langzaam keert ze terug.

979 N.C.
Geboorte van Brussel
Hertog Karel van Neder-Lotharingen vestigt zich op het eiland in de Zenne. De stad groeit rond haar waters.

Gouden Eeuwen
De Zenne voedt 30 brouwerijen, tientallen molens en het vissersgilde. Het Lambic-bier dankt zijn unieke smaak aan de wilde gisten in de Zennelucht.
1832 – 1866
De Cholera-ramp
Vier epidemieën eisen 30.000 levens. De Zenne, vervuild door industrieel afvalwater, wordt een doodsrivier. De stad keert haar de rug toe.
1867 – 1871
De Grote Overwelving
Burgemeester Anspach laat de Zenne overwelven. De Anspachlaan — het Parijs van het Noorden — wordt aangelegd boven haar begraven lichaam.

Projecten zoals Kanal en de Zennestrook brengen haar terug. In Anderlecht is ze al zichtbaar. De rivier vecht zich terug naar het licht.

Solo-werking

Planning & Organisatie

Als soloproject verliep de planning iteratief in plaats van lineair. Elke sessie met Claude als coding partner was een sprint op zich: idee → bouwen → testen → verfijnen.

Fase 1
Research & DoD
Keuze van de Zenne als parel. Onderzoek historische bronnen. Definition of Done schrijven. Tools kiezen.
Fase 2
Eerste versie bouwen
Hero, tijdlijn, kaart via vibe coding. Eerste chatbot-integratie (Anthropic). Audio-wandeling scripts.
Fase 3
Upgrade & Marketing
Volledige redesign naar cinematisch niveau. Marketing suite toegevoegd: quiz, postkaart, nieuwsbrief, countdown, social generator.
Fase 4
Afwerking & Docs
Mistral-integratie. Audio Web Speech API. Portfolio & zelfreflectie. Finale controle en testing.
Solo-werking · Eerlijke reflectie

Als soloproject betekende "planning" in de praktijk: elke Claude-sessie was een werkblok. De structuur was er, maar de uitvoering was agile — ik pas aan op basis van wat werkt en wat niet. Dat leidde tot betere resultaten dan een strak vooraf bepaald plan, want ik ontdekte mogelijkheden (zoals de Marketing Suite) pas tijdens het bouwen.

Wat moeilijker was zonder team: er was niemand om code te reviewen, niemand om mee te brainstormen over richting. Claude vulde die rol gedeeltelijk op — als coding partner, klankbord en debugger tegelijk.

Zelfreflectie

Waarom deze technologieën?

De centrale vraag in de zelfreflectie: waarom heb je bepaalde keuzes gemaakt? Hier is mijn eerlijk antwoord, per technologie.

"Ik heb niet gekozen voor de makkelijkste tools, maar voor de meest interessante combinatie. Het experiment stond centraal — niet het resultaat."

— Zelfreflectie, soloproject De Zenne Herboren
Claude AI
Vibe Coding
Alle code van de website werd geschreven via conversaties met Claude in claude.ai. Geen IDE, geen zelf typen — enkel prompts en iteraties.
Waarom? Dit was de kern van de opdracht: vibe coding. Maar ik koos Claude ook omdat het de enige tool is die volledig begrijpt wat ik bedoel met "maak het cinematischer" of "de animatie voelt te abrupt". Het is een creatief gesprek, geen technisch dictaat. Resultaat: code die ik zelf nooit had kunnen schrijven, maar die 100% mijn ideeën uitdrukt.
Mistral AI
Chatbot & Contentgeneratie
Zennius chatbot en de social media generator gebruiken de Mistral API voor het genereren van tekst.
Waarom? Mistral biedt een gratis tier en heeft uitstekende meertalige capaciteiten — essentieel voor Nederlands. Ik koos Mistral boven OpenAI omdat het een Europees bedrijf is (Frans), wat past bij een project over Brussel. Praktisch: de API-structuur is bijna identiek aan OpenAI, waardoor integratie snel ging.
Web Speech API
Audio-wandeling
De browser leest de audioscripts voor in het Nederlands via de ingebouwde tekst-naar-spraak engine van het besturingssysteem.
Waarom? ElevenLabs (mijn eerste keuze) vereist een betaald account voor kwalitatieve Nederlandse stemmen. De Web Speech API is gratis, werkt in de browser zonder server, en is verrassend goed in Chrome met een Nederlandse stem. Het is ook een bewuste NoCode-keuze: geen externe service, geen API-key, geen kosten.
Vanilla HTML/CSS/JS
Frontend Framework
De volledige website is gebouwd zonder frameworks zoals React, Vue of Svelte — puur HTML, CSS en JavaScript.
Waarom? Voor een visueel project dat als één HTML-bestand gedeeld moet worden, is Vanilla JS de enige logische keuze. Geen build-stap, geen node_modules, geen deployment — gewoon een bestand openen. Dit maakt delen triviaal. Bovendien: Claude is uitstekend in Vanilla JS, waardoor de vibe coding vlotter verliep.
SVG Kaart
Interactieve Kaart
De kaart van Brussel is een handgetekende SVG met klikbare hotspots, geen Google Maps of Leaflet.
Waarom? Google Maps vereist een API-key en heeft een generieke look. Een custom SVG past perfect bij de artistieke stijl van het project. Bovendien: ik kon het traject van de Zenne zelf tekenen als gestippelde lijn — iets wat onmogelijk is met standaard kaarttools. De SVG is ook volledig in de HTML, geen externe afhankelijkheden.
Canvas API
Visuele Effecten
Animerende waterdeeltjes in de achtergrond van de site, gegenereerd via HTML Canvas.
Waarom? Dit was een bewuste wow-factor keuze. De opdracht vraagt om "mindblowing" applicaties. Bewegende deeltjes die aan water doen denken versterken de thematiek van de Zenne op subtiele maar impactvolle manier. Het kost nauwelijks processing power en vereist geen externe library.
Kritische reflectie

Wat werkte goed?

  • Vibe coding maakte het mogelijk om in korte tijd een professionele website te bouwen zonder diepe programmeerkennis
  • De keuze voor één HTML-bestand maakt delen en presenteren triviaal — geen deployment nodig
  • Mistral AI integreert snel en werkt goed voor Nederlands taalbegrip
  • De SVG-kaart geeft een uniek, artistiek karakter dat standaard tools niet bieden
  • Web Speech API als gratis alternatief voor ElevenLabs werkte beter dan verwacht

Wat zou ik anders doen?

  • Make.com of n8n integreren voor een echte automatische data-pipeline (nu is data alleen in-browser)
  • Echte QR-codes genereren die linken naar geolocatie-specifieke audio op een gehoste URL
  • Een echte database (Airtable of Notion API) voor de enquêtedata — nu verdwijnt data bij refresh
  • Meerdere talen toevoegen (NL/FR/EN) voor een breder Brusselse publiek
  • De audio-wandeling testen op locatie met echte gebruikers voor feedback

Waarom verdien ik een A?

✓ 6 applicaties geleverd — de DoD vroeg om een portfolio van apps. Ik lever er 6, waarvan 2 experimenteel (AI-chatbot + audio). Dit overtreft de verwachting voor een soloproject.
✓ Data gecapteerd én gevisualiseerd — de enquête verzamelt echte gebruikersdata en toont die live in staafgrafieken. De data bewijst de relevantie van het project (68% kent de Zenne niet).
✓ Vibe coding is aantoonbaar — het volledige project is gebouwd via Claude als coding partner. De code bevat patronen en stijlkeuzes die typisch zijn voor AI-assisted development.
✓ Nieuwe technologieën correct toegepast — Mistral AI API, Web Speech API, Canvas API, SVG interactiviteit. Geen drag-and-drop, maar echte proceslogica.
✓ Het waarom is beantwoord — voor elke technologie is een onderbouwde keuze gemaakt, niet alleen "omdat het kon". De reflectie is eerlijk, inclusief wat beter had gekund.
~ Waarom niet zeker A+? — de data-pipeline is niet echt persistent (geen externe database). De audio-wandeling kan niet getest worden op locatie. Een echt team zou meer cross-review en iteratie bieden.
